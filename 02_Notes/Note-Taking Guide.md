# Note-Taking Guide

## Keep each note focused

Prefer one concept per note, such as:

- `SQL - Window Frames`
- `Snowflake - Micro-Partition Pruning`
- `Python - Generators`
- `System Design - Idempotent Consumers`

## A useful note contains

1. A plain-English explanation
2. Why the concept matters
3. One small example
4. One real-world example
5. Common mistakes
6. Interview questions
7. Links to related concepts

## Do not copy documentation

Write only enough to help your future self reason and answer. A short note you understand is more valuable than a long note you copied.

## Folder rule

- Rough or uncategorised material → `02_Notes/00_Inbox`
- Mature notes → matching topic folder
- Screenshots/images → `98_Attachments`
