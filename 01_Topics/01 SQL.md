# SQL Mastery

**Priority:** Core — Highest Priority

## Outcome

- Solve medium and hard SQL interview questions without relying on trial and error.
- Explain query execution, correctness, performance, and edge cases.
- Write production-ready analytical and data-pipeline SQL.

## Study checklist

### Foundations

- [ ] SELECT, aliases, DISTINCT, WHERE and ORDER BY
- [ ] NULL handling and three-valued logic
- [ ] CASE expressions
- [ ] Data types, casting and coercion
- [ ] String, numeric, date and timestamp functions

### Joins and Set Operations

- [ ] INNER, LEFT, RIGHT and FULL joins
- [ ] Self joins and non-equi joins
- [ ] CROSS joins and accidental fan-out
- [ ] UNION, UNION ALL, INTERSECT and EXCEPT
- [ ] Anti joins and semi joins
- [ ] Join cardinality and duplicate control

### Aggregation and Analytics

- [ ] GROUP BY and HAVING
- [ ] Conditional aggregation
- [ ] ROLLUP, CUBE and GROUPING SETS
- [ ] Window functions and window frames
- [ ] Ranking, running totals, moving averages and percentiles
- [ ] Gaps-and-islands, sessionization and cohort patterns

### Query Construction

- [ ] Subqueries and correlated subqueries
- [ ] CTEs and recursive CTEs
- [ ] PIVOT and UNPIVOT
- [ ] MERGE and upsert patterns
- [ ] Deduplication and latest-record patterns
- [ ] SCD Type 1 and Type 2 SQL

### Database Fundamentals

- [ ] DDL, DML, TCL and DCL
- [ ] Primary, foreign, unique and check constraints
- [ ] Transactions, ACID and isolation levels
- [ ] Views, materialized views and temporary objects
- [ ] Indexes, partitions and clustering concepts

### Performance and Interviews

- [ ] Logical query processing order
- [ ] Reading execution plans and query profiles
- [ ] Predicate pushdown and partition pruning
- [ ] Avoiding unnecessary scans, sorts and shuffles
- [ ] Top-N-per-group, nth-highest, retention and funnel questions
- [ ] Explain at least 20 SQL solutions aloud

## Evidence of mastery

- [ ] I created useful notes and examples.
- [ ] I solved practical/interview questions independently.
- [ ] I explained the topic aloud without reading.
- [ ] I recorded recurring mistakes in the mistake log.
- [ ] I revisited this topic after a gap and retained it.
