# Snowflake Mastery

**Priority:** Core — Highest Priority

## Outcome

- Design and operate reliable Snowflake data platforms.
- Diagnose performance and cost issues using Snowflake tooling.
- Answer architecture, security and pipeline interview questions confidently.

## Study checklist

### Architecture and Objects

- [ ] Cloud services, storage and virtual warehouse layers
- [ ] Databases, schemas, tables, views and secure views
- [ ] Permanent, transient and temporary objects
- [ ] Micro-partitions and metadata
- [ ] Zero-copy cloning and Time Travel
- [ ] Fail-safe concepts and limitations

### Loading and Unloading

- [ ] Internal and external stages
- [ ] File formats and storage integrations
- [ ] COPY INTO table and validation modes
- [ ] COPY INTO location for unloading
- [ ] Snowpipe and event-driven ingestion
- [ ] Handling bad records, schema drift and duplicate files

### Transformations and Automation

- [ ] Streams and change tracking
- [ ] Tasks and task graphs
- [ ] Dynamic tables and target lag
- [ ] Stored procedures, UDFs and Snowpark basics
- [ ] MERGE and incremental processing
- [ ] Idempotent pipeline design

### Semi-Structured Data

- [ ] VARIANT, OBJECT and ARRAY
- [ ] Parsing JSON and path navigation
- [ ] FLATTEN and lateral joins
- [ ] Schema-on-read patterns
- [ ] Performance considerations for nested data

### Security and Governance

- [ ] RBAC hierarchy and least privilege
- [ ] Future grants and managed access schemas
- [ ] Network policies and authentication options
- [ ] Masking policies and row access policies
- [ ] Tags, classification and access history
- [ ] Secure data sharing and marketplace concepts

### Performance, Cost and Operations

- [ ] Warehouse sizing, scaling and auto-suspend
- [ ] Query Profile and operator analysis
- [ ] Clustering, automatic clustering and pruning
- [ ] Search Optimization Service and materialized views
- [ ] Caching layers and result reuse
- [ ] Resource monitors, budgets and cost attribution
- [ ] ACCOUNT_USAGE vs INFORMATION_SCHEMA
- [ ] Replication, failover and disaster-recovery concepts
