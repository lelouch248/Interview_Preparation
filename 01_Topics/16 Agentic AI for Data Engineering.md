# Agentic AI for Data Engineering

**Priority:** Advanced / Differentiator

## Outcome

- Build constrained agents that perform useful engineering work.
- Use tools, state and evaluation instead of vague autonomous behavior.
- Apply AI safely to Snowflake and data operations.

## Study checklist

### LLM Foundations

- [ ] Tokens, context windows and sampling
- [ ] Prompt structure and structured outputs
- [ ] Embeddings and retrieval basics
- [ ] Tool/function calling
- [ ] Model limitations and hallucination control

### Agent Architecture

- [ ] Single agent vs workflow vs multi-agent
- [ ] Planner, executor and reviewer patterns
- [ ] State, memory and checkpoints
- [ ] Tool permissions and read-only boundaries
- [ ] Human approval and escalation
- [ ] LangGraph concepts

### Data Engineering Use Cases

- [ ] Schema and DDL drift detection
- [ ] Failed-task triage
- [ ] Mapping-sheet validation
- [ ] Query optimization suggestions
- [ ] Data-quality investigation
- [ ] Documentation and lineage assistance

### Production Safety

- [ ] Deterministic checks before LLM analysis
- [ ] Evaluation datasets and expected outputs
- [ ] Prompt/version tracking
- [ ] Audit logs and explainability
- [ ] Cost, latency and caching
- [ ] Security, PII and credential isolation

### Portfolio Practice

- [ ] Build one read-only Snowflake agent
- [ ] Add a rules engine before the LLM
- [ ] Create a reviewer/approval step
- [ ] Measure accuracy and false positives
- [ ] Present limitations honestly
