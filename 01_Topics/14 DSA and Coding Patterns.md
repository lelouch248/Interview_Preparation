# DSA & Coding Patterns

**Priority:** Interview Skill

## Outcome

- Clear coding rounds expected for data/software engineering roles.
- Recognize common patterns instead of memorizing isolated problems.
- Communicate complexity and edge cases.

## Study checklist

### Core Structures

- [ ] Arrays and strings
- [ ] Hash maps and sets
- [ ] Stacks and queues
- [ ] Linked lists
- [ ] Heaps and priority queues
- [ ] Trees and graphs basics

### Core Patterns

- [ ] Two pointers
- [ ] Sliding window
- [ ] Prefix sums
- [ ] Binary search
- [ ] Intervals
- [ ] BFS and DFS
- [ ] Top-K and heap patterns
- [ ] Basic dynamic programming

### Interview Execution

- [ ] Clarify input and constraints
- [ ] Start with brute force
- [ ] Improve with the right data structure
- [ ] State time and space complexity
- [ ] Test normal, empty, duplicate and boundary cases
- [ ] Write readable Python under time pressure

### Recommended Scope

- [ ] Master easy problems
- [ ] Become consistent on common medium problems
- [ ] Prioritize data-oriented problems over competitive programming
- [ ] Maintain a mistake log instead of chasing problem count

## Evidence of mastery

- [ ] I created useful notes and examples.
- [ ] I solved practical/interview questions independently.
- [ ] I explained the topic aloud without reading.
- [ ] I recorded recurring mistakes in the mistake log.
- [ ] I revisited this topic after a gap and retained it.
