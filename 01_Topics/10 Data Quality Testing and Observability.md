# Data Quality, Testing & Observability

**Priority:** Production Skill

## Outcome

- Define measurable data-quality rules.
- Build layered validation and monitoring.
- Respond to incidents using evidence and clear ownership.

## Study checklist

### Quality Dimensions

- [ ] Completeness
- [ ] Validity
- [ ] Uniqueness
- [ ] Consistency
- [ ] Timeliness and freshness
- [ ] Accuracy and reconciliation

### Testing Layers

- [ ] Schema and contract tests
- [ ] Source-to-target row-count checks
- [ ] Aggregate and control-total reconciliation
- [ ] Business-rule tests
- [ ] Referential-integrity checks
- [ ] Regression and anomaly detection

### Observability

- [ ] Pipeline health and run metadata
- [ ] Freshness and volume monitoring
- [ ] Schema-change detection
- [ ] Lineage and impact analysis
- [ ] SLIs, SLOs and incident thresholds
- [ ] Alert deduplication and severity

### Framework Design

- [ ] Rule configuration and metadata tables
- [ ] Reusable validators
- [ ] Result storage and evidence
- [ ] Failure quarantine
- [ ] Dashboards and ownership
- [ ] Automated remediation boundaries

### Interview Practice

- [ ] Design a validation framework
- [ ] Handle a false-positive alert
- [ ] Detect silent data corruption
- [ ] Define an incident runbook
