# dbt Mastery

**Priority:** Core

## Outcome

- Build modular, tested and documented transformations.
- Design dbt projects that scale across teams and environments.
- Explain incremental models, snapshots and CI/CD.

## Study checklist

### Core Workflow

- [ ] Project structure and profiles
- [ ] Models, ref and source
- [ ] Materializations: view, table, incremental and ephemeral
- [ ] Seeds and snapshots
- [ ] Generic and singular tests
- [ ] Documentation and lineage

### Jinja and Reuse

- [ ] Variables and environment-aware logic
- [ ] Macros and dispatch
- [ ] Loops, conditionals and reusable SQL
- [ ] Packages and dependency management
- [ ] Hooks and operations

### Advanced Modeling

- [ ] Incremental strategies and unique keys
- [ ] Schema changes in incremental models
- [ ] Snapshots and SCD Type 2
- [ ] Custom tests and test severity
- [ ] Exposures, metrics and semantic-layer concepts
- [ ] Contracts and model versions

### Production Practices

- [ ] Folder and naming conventions
- [ ] Sources freshness
- [ ] Slim CI and state comparison
- [ ] Deferral and environment promotion
- [ ] Artifacts: manifest, run results and catalog
- [ ] Deployment jobs, alerts and troubleshooting

### Interview Practice

- [ ] Design a dbt project for multiple domains
- [ ] Debug a failing incremental model
- [ ] Choose between snapshot and merge logic
- [ ] Explain dbt vs stored-procedure transformations
