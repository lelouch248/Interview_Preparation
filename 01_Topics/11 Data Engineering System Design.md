# Data Engineering System Design

**Priority:** Core Interview Skill

## Outcome

- Turn vague requirements into a complete architecture.
- Estimate scale and identify bottlenecks.
- Explain trade-offs, failure modes, security and cost.

## Study checklist

### Requirement Discovery

- [ ] Functional and non-functional requirements
- [ ] Data volume, velocity and retention
- [ ] Freshness and latency targets
- [ ] Consumers and access patterns
- [ ] Compliance, residency and recovery targets

### Architecture Building Blocks

- [ ] Source ingestion
- [ ] Message queues and streaming
- [ ] Object storage and lake layers
- [ ] Processing engines
- [ ] Warehouse/lakehouse serving
- [ ] Orchestration, catalog and observability

### Design Qualities

- [ ] Scalability and partitioning
- [ ] Reliability and replay
- [ ] Consistency and ordering
- [ ] Idempotency and deduplication
- [ ] Security and least privilege
- [ ] Cost and operational simplicity

### Common Designs

- [ ] Batch analytics platform
- [ ] Near-real-time event pipeline
- [ ] CDC replication platform
- [ ] Multi-tenant data platform
- [ ] Data-quality control tower
- [ ] Metadata and lineage system

### Interview Method

- [ ] Clarify requirements first
- [ ] State assumptions and estimates
- [ ] Draw end-to-end flow
- [ ] Deep dive into two risky components
- [ ] Cover failure modes, monitoring and trade-offs
- [ ] Summarize alternatives and evolution path
