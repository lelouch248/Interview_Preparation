# Python for Data Engineering

**Priority:** Core — Highest Priority

## Outcome

- Write clean, testable Python for pipelines and automation.
- Solve interview coding questions with clear reasoning.
- Understand performance, reliability and packaging basics.

## Study checklist

### Language Foundations

- [ ] Variables, types, operators and control flow
- [ ] Lists, tuples, sets and dictionaries
- [ ] Slicing, comprehensions and unpacking
- [ ] Functions, scope, args and kwargs
- [ ] Mutability, identity, equality and copying

### Intermediate Python

- [ ] Classes, inheritance, composition and dataclasses
- [ ] Exceptions and custom exception design
- [ ] Iterators and generators
- [ ] Decorators and closures
- [ ] Context managers
- [ ] Type hints, protocols and generics basics

### Engineering Practices

- [ ] Modules, packages and virtual environments
- [ ] Logging and configuration management
- [ ] Unit testing with pytest
- [ ] Mocking external systems
- [ ] Linting, formatting and static checking
- [ ] Dependency management and reproducible environments

### Data Engineering Python

- [ ] CSV, JSON, Parquet and file handling
- [ ] Database connectivity and parameterized queries
- [ ] REST API consumption and pagination
- [ ] Batching, retries, backoff and idempotency
- [ ] Pandas transformations and memory awareness
- [ ] Schema validation and data-quality checks

### Performance and Concurrency

- [ ] Big-O for common Python structures
- [ ] Profiling CPU and memory
- [ ] Threading vs multiprocessing vs asyncio
- [ ] I/O-bound vs CPU-bound workloads
- [ ] Streaming large files instead of loading everything

### Interview Patterns

- [ ] Strings, arrays and hash maps
- [ ] Stacks, queues and heaps
- [ ] Two pointers and sliding window
- [ ] Sorting, binary search and intervals
- [ ] Recursion and basic trees
- [ ] Explain trade-offs and test edge cases aloud

## Evidence of mastery

- [ ] I created useful notes and examples.
- [ ] I solved practical/interview questions independently.
- [ ] I explained the topic aloud without reading.
- [ ] I recorded recurring mistakes in the mistake log.
- [ ] I revisited this topic after a gap and retained it.
