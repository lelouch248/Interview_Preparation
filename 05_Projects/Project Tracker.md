## A project is interview-ready when it has

- [ ] Clear business problem
- [ ] Requirements and constraints
- [ ] Architecture diagram
- [ ] Data flow and component responsibilities
- [ ] Failure handling and idempotency
- [ ] Testing and observability
- [ ] Security and access model
- [ ] Performance and cost decisions
- [ ] Measured or defensible impact
- [ ] Honest limitations and next version
- [ ] A 2-minute and a 10-minute explanation
